Ongoing Personal Project

Introduction

A series of 2D/3D tests in JavaScript for a 4D game I am working on based on the games Miegakure and Tetraspace. The plan for the game is for the player to solve puzzles within a world with 4 spatial dimensions. The player can move as normal in a 3d plane but has the ability to rotate this plane 90 degrees around them, allowing them to access different parts of the world.

I chose JavaScript as I needed an object oriented language that I had some experience with and could easily draw lines to the screen with. However, I aim to move over to either C++ (either using middleware, or as part of the Unreal Engine) or C# (as part of Unity) when moving to a 3D/4D space.

Current Progress

Version 1: Cross-Section of a 3D Mesh
The core mechanic of the game. This calculates and renders a 2D cross-section of a 3D scene. This allowed me to get an understanding of the maths required for the project in a simpler scenario (one fewer dimension) with shapes I could more easily visualise.

Version 2: 2D Game Engine
A 2D game engine in JavaScript. My aim is to move into a 3D game engine, most of which represent the game objects using meshes formed of vertices. Most 2D game engines, however, use sprites, so I needed a 2D engine which used vertex meshes which I could easily modify.

Version 3: Combination
A platformer game which uses both the 2D Game Engine and the core rotation mechanic, so that I could work out how to combine the above two versions and incorporate the mechanic within a game engine.

Plans in the Future

  • Improve Code Quality
  • Extend the collision detection within the prototype. Currently the game only performs collision detection along the visible plane. I Would want to have a greater degree of 3d collision detection, allowing the player to move blocks which extend through multiple planes.
  • Create a 3d/4d version within C++/Unity/Unreal of the core mechanic. Most likely to use an existing engine. Although I would be more limited within the engine, I would have access to all the tools and existing code that engine would provide. It may also help with porting the game to other platforms/engines if I am able to have the game working within a 3D game engine.
