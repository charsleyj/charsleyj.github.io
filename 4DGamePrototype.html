4D Game Prototype

Ongoing Personal Project

Introduction

This is a prototype for a 4D game which I am working on. Based on the games Miegakure and Brane (previously known as Tetraspace), the plan for the game is for the player to solve puzzles within a world with 4 spatial dimensions. The player can move as normal in a 3d "hyperplane" but has the ability to rotate this "hyperplane" 90 degrees around them, allowing them to access different parts of the world.

At the moment, this is a simple 2D/3D prototype in JavaScript. This was chosen as I needed an object oriented language that I had some experience with and could easily draw lines to the screen with. However, I aim to move over to either C++ (either using middleware, or as part of the Unreal Engine) or C# (as part of Unity) when moving to 3D/4D space.

Current Progress

The prototype currently consists of 3 versions:

  • The first is a small program testing the core mechanic of the game. The program calculates and renders the 2D cross-section of a 3D scene. This allowed me to get an understanding of the maths required for the project in a simpler scenario (1 fewer dimensions) with shapes I could more easily visualise in my head.
  • The second is a very simple 2d game engine in JavaScript. My aim is to move into a 3D game engine, most of which represent the game objects using meshes formed of vertices. Most 2D game engines, however, use sprites. So I needed a 2D engine which used vertex meshes which I could easily modify.
  • In the third, I combined them both to create a simple platformer with the core rotation mechanic, so that I could work out how to incorporate the mechanic within a game engine.

Plans in the Future

  • Improve Code Quality
  • Extend the collision detection within the prototype. Currently the game only performs collision detection along the visible plane. I Would want to have a greater degree of 3d collision detection, allowing the player to move blocks which extend through multiple planes.
  • Create a 3d/4d version within C++/Unity/Unreal of the core mechanic. Most likely to use an existing engine. Although I would be more limited within the engine, I would have access to all the tools and existing code that engine would provide. It may also help with porting the game to other platforms/engines if I am able to have the game working within a 3D game engine.
